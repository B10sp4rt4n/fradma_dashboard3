```
ZIP con XMLs  →  Parser CFDI 4.0  →  Base de datos cloud  →  Clientes auto-extraídos
                                                                        ↓

```

Cada paso es automático. El usuario sube un ZIP. El sistema hace el resto.

---

## Lo que hace internamente

**Procesamiento de CFDIs**
- Extrae y parsea cientos de XMLs CFDI 4.0 en segundos — incluyendo los de proveedores y terceros, no solo los que la empresa emite en su propio sistema
- Maneja errores, archivos corruptos y codificación especial sin intervención del usuario
- Almacena en base de datos estructurada con soporte multi-empresa

**Padrón de clientes auto-construido**
- Cada factura alimenta automáticamente: RFC, razón social, régimen fiscal, domicilio, total histórico, primera y última fecha de venta
- Ningún CRM en México construye esto desde los CFDIs — siempre son mundos separados
- Resultado: **CRM fiscal automático sin captura manual**

## Por qué el CFDI como fuente de verdad

Los reportes del ERP reflejan lo que se capturó. El CFDI refleja lo que el SAT selló.

En México, el CFDI es el documento fiscal legalmente vinculante. CIMA lo toma como input primario: los datos son **100% reconciliables con el SAT**, sin depender de que el ERP esté bien alimentado. Una sola fuente de verdad para ventas, cobranza y declaraciones.
